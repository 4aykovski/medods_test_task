## Test task BackDev MEDODS

**Используемые технологии:**

- Go
- JWT
- MongoDB

**Задание:**

Написать часть сервиса аутентификации.

Два REST маршрута:

- Первый маршрут выдает пару Access, Refresh токенов для пользователя с идентификатором (GUID) указанным в параметре
  запроса
- Второй маршрут выполняет Refresh операцию на пару Access, Refresh токенов

**Требования:**

Access токен тип JWT, алгоритм SHA512, хранить в базе строго запрещено.

Refresh токен тип произвольный, формат передачи base64, хранится в базе исключительно в виде bcrypt хеша, должен быть
защищен от изменения на стороне клиента и попыток повторного использования.

Access, Refresh токены обоюдно связаны, Refresh операцию для Access токена можно выполнить только тем Refresh токеном
который был выдан вместе с ним.

**Результат:**

Результат выполнения задания нужно предоставить в виде исходного кода на Github.

## Структура

В реализации используется только стандартная библиотека, за исключением JWT и монго драйвера, из-за поставленных условий.

Всякие тесты отсутствуют из-за ограниченности стека технологий. Была попытка дополнительно об этом узнать, но ответа не
поступило.

- `cmd/app` - точка входа в приложение. Производится подготовка всех слоев и запуск сервера
- `internal`
    - `config` - структура конфига приложения.
    - `model` - описание моделей
    - `repository` - различные реализации репозиториев. В `repository.go` - общие ошибки, которые могут вернуться из
      репозиториев
        - `mongorepos` - реализация монго репозиториев
    - `rest` - версии REST API
        - `v1`
            - `handler` - хранение всех хендлеров для первой версии API
            - `middleware` - хранение миддлвееров для первой версии API
            - `router.go` - создание роутера и инициализация путей
    - `services` - реализация различных сервисов, которые управляют моделями
- `pkg`
    - `database`
        - `mongodb` - реализация подключения к монго
    - `lib`
        - `api`
            - `response` - базовая реализация всех респонсов
        - `hasher` - реализации хешеров
        - `auth` - реализации менджеров, который создает токены
- `tests` - функциональные тесты
